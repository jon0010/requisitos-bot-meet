Requisitos para automatizar entrevistas en Google Meet con un bot

Checklist de requisitos técnicos y permisos para el desarrollo de la aplicación.

1. Google Workspace con dominio propio

La automatización requiere un Workspace con dominio propio. No funciona con cuentas personales de Gmail.

Una suscripción de Google Workspace (Business Standard, Business Plus o Enterprise recomendado)

Más información: workspace.google.com

2. Acceso como Super Admin

El Super Admin debe otorgar permisos para activar APIs, crear Service Accounts y configurar delegaciones.

*Si el dueño del proyecto crea el Workspace y es propietario, será automáticamente Super Admin.

Documentación oficial: Roles de administrador en Google Workspace

3. Activación de APIs

  • Google Calendar API: https://developers.google.com/calendar/api
  • Google Meet API: https://developers.google.com/meet
  • Google Meet Media API (Developer Preview): https://developers.google.com/meet/media

4. Registro en Developer Preview Program

Necesario para usar la Meet Media API*. El proceso de aprobación puede tardar varios días.

* Google Meet Media API → permite a un bot enviar/recibir audio y video. Requiere estar en el Google Workspace Developer Preview Program. El superadmin debe inscribirse en Developer Preview y esperar la aprobación (pueden ser días).

Formulario: https://developers.google.com/workspace/preview

5. Creación y configuración de Service Account

La Service Account* permite que la aplicación actúe en nombre de usuarios del Workspace con permisos delegados.

* Una Service Account es una identidad especial para servidores/aplicaciones, no para usuarios humanos. Sirve para que tu backend actúe como si fuera un usuario del Workspace, pero de forma programática y con permisos delegados.

Guía para delegación de todo el dominio: https://developers.google.com/admin-sdk/directory/v1/guides/delegation

6. Configuración de privilegios para la Service Account

El Super Admin debe asignar scopes y permitir que la Service Account sea organizadora de reuniones.

Detalles: https://support.google.com/a/answer/162106

Checklist rápido

  • Google Workspace con dominio propio
  • Super Admin activo
  • APIs activadas: Calendar, Meet, Meet Media
  • Registro en Developer Preview (si aplica)
  • Service Account con domain-wide delegation
  • Privilegios y scopes asignados
